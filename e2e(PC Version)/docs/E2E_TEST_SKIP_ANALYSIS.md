# E2E 테스트 스킵 분석

## 스킵된 테스트의 원인

스킵된 테스트들은 **실패한 테스트들의 영향이 아닙니다**. 대부분은 다음과 같은 이유로 스킵되었습니다:

---

## 1. 명시적으로 스킵된 테스트 (기존 성공 테스트)

다음 테스트들은 기존에 성공했기 때문에 `test.skip()`으로 명시적으로 스킵 처리되었습니다:

1. ✅ `1-1-5: ログアウト` - `test.skip('1-1-5: ログアウト', ...)`
2. ✅ `1-2-1: キャラクター検索` - `test.skip('1-2-1: キャラクター検索', ...)`
3. ✅ `1-2-2: キャラクター一覧確認` - `test.skip('1-2-2: キャラクター一覧確認', ...)`
4. ✅ `1-7-1: 通知一覧確認` - `test.skip('1-7-1: 通知一覧確認', ...)`
5. ✅ `1-7-4: 未読通知数確認` - `test.skip('1-7-4: 未読通知数確認', ...)`
6. ✅ `1-9-3: 自分のお問い合わせ/通報一覧確認` - `test.skip('1-9-3: ...', ...)`
7. ✅ `1-10-1: ペルソナ一覧確認` - `test.skip('1-10-1: ペルソナ一覧確認', ...)`
8. ✅ `1-5-1: ポイント残高確認` - `test.skip('1-5-1: ポイント残高確認', ...)`
9. ✅ `1-8-2: 日次/週次/月次ランキング確認` - `test.skip('1-8-2: ...', ...)`

**이유**: 이미 성공한 테스트이므로 재실행에서 제외하여 시간 절약

---

## 2. 조건부 스킵 (테스트 실행 중 조건 미충족)

다음 테스트들은 실행 중 조건을 만족하지 못해 `test.skip(true, 'reason')`으로 스킵되었습니다:

### 캐릭터 관련
- `1-3-3: キャラクター編集` - 편집 가능한 캐릭터를 찾지 못함
- `1-3-5: キャラクタークローン` - 클론 버튼을 찾지 못함
- `1-5-4: ポイント使用確認` - 캐릭터를 찾지 못함 (하지만 이 테스트는 실행되어 실패함)
- `1-6-7: いいね機能` - 캐릭터를 찾지 못함
- `1-6-8: コメント作成` - 캐릭터를 찾지 못함
- `1-9-2: キャラクター通報` - 캐릭터 또는 통보 버튼을 찾지 못함

### 페르소나 관련
- `1-10-3: ペルソナ編集` - 편집 가능한 페르소나를 찾지 못함
- `1-10-4: ペルソナ削除` - 삭제 가능한 페르소나를 찾지 못함

### 기타
- `1-5-3: 出席チェック` - 출석 체크 버튼을 찾지 못함
- `1-7-2: 通知既読処理` - 미읽 알림을 찾지 못함
- `1-6-3: フォロー/アンフォロー` - 팔로우 버튼을 찾지 못함
- `1-11-2: お知らせ詳細確認` - 공지사항을 찾지 못함
- `1-1-6: パスワード変更` - 패스워드 변경 기능을 찾지 못함

**이유**: 테스트 실행 중 필요한 요소(캐릭터, 버튼, 기능 등)를 찾지 못해 스킵

---

## 3. 실패한 테스트와의 관계

### 실패한 테스트들 (실제로 실행됨)

다음 테스트들은 **실제로 실행되었지만 실패**했습니다:

1. ❌ `1-1-1: メールアドレス 회원가입` - **실행됨, 실패**
2. ❌ `1-11-4: 約款確認` - **실행됨, 실패**
3. ❌ `1-5-4: ポイント使用確認` - **실행됨, 실패** (캐릭터는 찾았지만 전송 버튼 실패)
4. ❌ `1-8-1: リアルタイムランキング確認` - **실행됨, 실패**
5. ❌ `1-6-1: 自分のプロフィール確認` - **실행됨, 실패**
6. ❌ `1-6-3: フォロー/アンフォロー` - **실행됨, 실패**

### 스킵된 테스트들

스킵된 테스트들은:
- **명시적으로 스킵된 테스트**: 기존 성공 테스트 (재실행 제외)
- **조건부 스킵**: 테스트 실행 중 조건 미충족 (요소를 찾지 못함)

---

## 결론

**스킵된 테스트들은 실패한 테스트들의 영향이 아닙니다.**

1. **명시적 스킵**: 기존 성공 테스트를 재실행에서 제외하기 위해 의도적으로 스킵
2. **조건부 스킵**: 테스트 실행 중 필요한 요소를 찾지 못해 스킵
3. **실패한 테스트**: 실제로 실행되었지만 실패 (스킵되지 않음)

스킵된 테스트 중 일부는 조건부 스킵으로, 실제로는 테스트 환경이나 데이터 부족으로 인한 것일 수 있습니다. 이를 해결하려면:
- 테스트 데이터 준비
- 선택자 개선
- 테스트 조건 완화

# 약관 기반 필수 기능 정리

> **작성일**: 2025-01-27  
> **목적**: 이용약관 및 개인정보처리방침에 따라 사이트가 정상 작동하기 위해 필요한 기능들을 정리

## ⚠️ 중요 참고사항

이 문서는 **약관 요구사항**을 기반으로 작성되었지만, **실제 일본 사이트들의 관행**을 고려하여 수정되었습니다:

- **연령 확인**: 대부분의 사이트는 "13세 이상입니다" 체크박스만 사용. 보호자 동의 절차는 거의 구현하지 않음.
- **필터 우회 감지**: 생년월일만 입력하면 되므로 우회가 가능함. 실제로 우회를 막는 사이트는 거의 없음.
- **계정 정보 변경**: 이메일/전화번호 변경 기능은 필수가 아님. 전화번호 인증은 SMS 비용 발생.
- **크리에이터 프로그램**: 약관에 명시되어 있지만 현재 미시작 상태이므로 미구현으로 둬도 됨.

**실제 구현 시에는 법적 최소 요구사항과 비용을 고려하여 우선순위를 정하는 것이 중요합니다.**

## 1. 연령 확인 및 보호자 동의

### 📌 약관 요구사항
- **제6조**: 13세 이상만 이용 가능, 13-18세는 보호자 동의 필요
- **제7조**: 등록 시 본인 확인
- **제8조**: 13세 미만 또는 보호자 동의 없는 미성년자 등록 거부
- **제9조**: 연령 위조 시 계정 삭제

### ✅ 현재 구현 상태
- `dateOfBirth` 필드 존재 (schema.prisma)
- `declaredAdult` 필드 존재
- `adultVerificationDate`, `adultVerificationBirthdate` 필드 존재
- 연령 확인 페이지 일부 구현됨 (AGE_VERIFICATION_LEGAL_REQUIREMENTS.md 참고)

### ❌ 필요한 추가 기능

> **참고**: 실제 일본 사이트들의 관행을 고려하면, 대부분의 사이트는 간단한 체크박스 확인만 사용합니다. 아래 기능들은 법적 요구사항이지만, 실제 구현은 간소화할 수 있습니다.

#### 1.1 회원가입 시 연령 확인 (최소 요구사항)
- [ ] **"13세 이상입니다" 체크박스** (회원가입 시 필수)
  - 체크박스 선택 시에만 가입 가능
  - 체크박스 미선택 시 가입 차단
- [ ] **생년월일 입력** (선택사항, 권장)
  - 연령 계산 및 검증 (13세 미만 자동 거부)
  - 약관 위반 시 책임 회피를 위한 기록

#### 1.2 보호자 동의 (선택사항 - 법적 요구사항이지만 실제로는 거의 구현 안 함)
- [ ] **13-18세 보호자 동의 절차** (구현 시)
  - 보호자 이메일 입력
  - 보호자 동의 이메일 발송
  - 보호자 동의 링크 클릭 확인
  - 동의 상태 DB 저장
- ⚠️ **실제로는 대부분의 사이트가 보호자 동의 절차를 구현하지 않음**

#### 1.3 연령 위조 감지 (선택사항)
- [ ] **연령 위조 시도 감지 로직** (발견 시에만 대응)
- [ ] **위조 감지 시 계정 삭제** (약관 명시)
- ⚠️ **실제로는 생년월일만 입력하면 되므로 우회가 가능함. 대부분의 사이트가 적극적으로 막지 않음**

## 2. 계정 관리

### 📌 약관 요구사항
- **제7조**: 등록 시 본인 확인, 정확한 정보 제공
- **제10조**: 계정 정보 관리 책임, 공유/양도/대여/매매 금지
- **제11조**: 이용 환경 준비 책임
- **제34조**: 사용자 해지 기능

### ✅ 현재 구현 상태
- 회원가입 기능 구현됨
- 계정 삭제 기능 구현됨 (`/api/users/account` DELETE)
- 계정 정지 기능 구현됨 (`suspendedUntil`, `suspensionReason`)

### ❌ 필요한 추가 기능

#### 2.1 계정 정보 관리
- [ ] **계정 정보 수정 기능** (이메일, 전화번호 등)
  - ⚠️ **이메일 변경**: 필수 기능은 아님. 변경 시 재인증 절차 권장
  - ⚠️ **전화번호 변경**: 필수 기능은 아님. 전화번호 인증은 SMS 비용 발생
    - **SMS 비용**: 메시지당 약 7-9엔 (Twilio: $0.05, AWS SNS: $0.06)
    - 월 1,000건 기준 약 7,000-9,000엔, 월 10,000건 기준 약 70,000-90,000엔
    - 구현 시 비용 대비 효과를 고려하여 결정 필요
- [ ] **계정 공유/양도 감지 시스템** (동시 접속 감지 등)
  - ⚠️ **선택사항**: 복잡한 시스템이므로 우선순위 낮음

#### 2.2 계정 해지
- [ ] **30일 유예 기간 해지 시스템**
  - 해지 신청 시 즉시 삭제가 아닌 30일 후 삭제
  - 30일 내 해지 취소 가능
  - 해지 예정일 알림
- [ ] **해지 시 데이터 보존 기간 관리** (법적 요구사항에 따라)

## 3. 포인트 시스템

### 📌 약관 요구사항
- **제12조**: 유상 포인트, 무상 포인트 구분
- **제13조**: 포인트 구매 방법
- **제14조**: 무상 포인트 1년 유효기간, 유상 포인트 무기한
- **제15조**: 사용 순서 (무상 우선, 오래된 것부터)
- **제16조**: 현금화/환불 불가 (법령 제외)
- **제17조**: 불법 거래 대응
- **제18조**: 외부 결제 서비스 (Stripe) 연동

### ✅ 현재 구현 상태
- 포인트 시스템 기본 구현됨 (`free_points`, `paid_points`)
- Stripe 결제 연동 구현됨
- 무상 포인트 우선 소비 로직 구현됨
- 출석 체크로 무상 포인트 지급 구현됨

### ❌ 필요한 추가 기능

#### 3.1 포인트 유효기간 관리
- [ ] **무상 포인트 유효기간 필드 추가**
  - `points.freePointsExpiresAt` 필드 추가
  - 무상 포인트 지급 시 1년 후 만료일 설정
- [ ] **유효기간 만료 처리**
  - 만료된 무상 포인트 자동 삭제
  - 만료 7일 전 알림 발송
- [ ] **유상 포인트 무기한 보장** (현재 구현됨)

#### 3.2 포인트 사용 순서 관리
- [ ] **무상 포인트 우선 소비** (현재 구현됨 ✅)
- [ ] **지급일 오래된 순서로 소비**
  - 무상 포인트 지급 이력 테이블 필요
  - FIFO (First In First Out) 방식 구현
- [ ] **포인트 지급 이력 관리**
  - `point_transactions` 테이블 생성
  - 지급일, 만료일, 잔액 추적

#### 3.3 불법 거래 감지 및 대응
- [ ] **단시간 고액 구매 감지**
  - 1시간 내 10만엔 이상 구매 시 알림
- [ ] **신규 계정 고액 구매 감지**
  - 가입 후 24시간 내 5만엔 이상 구매 시 알림
- [ ] **동일 IP 다중 계정 감지**
  - 동일 IP에서 여러 계정 구매 시 알림
- [ ] **관리자 알림 시스템**
  - 의심 거래 자동 알림
  - 거래 취소 기능

#### 3.4 포인트 구매 제한
- [ ] **1일 구매 상한선** (10만엔)
- [ ] **계정 보유 상한선** (100만 포인트)
- [ ] **미성년자 월간 상한선** (1만엔)
